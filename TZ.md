# ТЗ — Realty Telegram Mini App (Дніпро)

## Концепция
Telegram Mini App для одного агентства недвижимости (пара риелторов, 2 жёстко прошитых TG ID).
- **Клиент**: смотрит, сохраняет, записывается на просмотр.
- **Риелтор**: полностью управляет объявлениями внутри Mini App, получает уведомления в боте.

---

## 1. Технологический стек

### Frontend
- Vite + React + **TypeScript**
- Перенос текущего дизайна из `Rieltor app/` (screens, components, styles)
- Поддержка темы Telegram (light/dark автоматически)
- Только украинский язык
- Хостинг: на VPS пользователя (Nginx + статика)

### Backend
- Node.js + **Fastify**
- **PostgreSQL** (на том же VPS)
- TypeScript
- Авторизация: валидация `Telegram.WebApp.initData` через токен бота, выдача JWT
- REST API

### Бот
- Node.js + **grammY**
- На том же VPS
- Канал: уведомления риелторам (2 TG ID) + опционально клиентам (напоминания)

### Хранилище
- **Фото объектов** — на сервере в папке `/uploads/objects/{id}/`
  - Максимум 50 фото на объект
  - Авто-сжатие через **sharp**: до 1920px по длинной стороне, WebP + JPEG fallback
  - Превью 400px для списков
- **Фото риелторов** — `/uploads/agents/{id}/`

### Карта
- **Leaflet** + слой **OpenStreetMap** (полностью бесплатно, только схема — без спутника)
- При добавлении объекта: риелтор кликает по карте → точка ставится → адрес пишет вручную (без автогеокодинга)

### Курс валют
- Фоновая задача (cron) **раз в сутки в 09:00** дёргает API НБУ
- Сохраняем в таблицу `exchange_rates`
- Используется для автоконвертации при показе цены и при создании объекта

### Развёртывание
- Docker Compose (db + api + bot + nginx) — одна команда запуска
- Автобэкап раз в сутки: дамп Postgres + tar папки uploads → локально на сервере
- HTTPS через Let's Encrypt (Certbot)
- Домен — пользователь купит позже

## 2. Роли

### Клиент (любой TG-юзер кроме риелторов)
- Просмотр объектов (лента, детали, галерея)
- Фильтры + карта
- Избранное
- Запись на просмотр
- Шеринг объекта в TG

### Риелтор (2 жёстко зашитых TG ID в `.env`)
- Всё, что может клиент
- + Кабинет: CRUD объектов, аналитика, лиды
- При первом запуске на сервере — TG ID и имена пары риелторов задаются в env

## 3. Структура данных

### Объект (`properties`)
**Обязательные поля:**
- `id`, `created_at`, `updated_at`, `created_by` (риелтор)
- `status`: `draft` | `active` | `reserved` | `sold_rented` | `withdrawn`
- `type`: `apartment` | `house` | `commercial` | `land`
- `deal`: `sale` | `rent`
- `price_value`, `price_currency` (USD | UAH)
- `price_value_secondary`, `price_currency_secondary` (автозаполнение по НБУ, риелтор может переписать)
- `address` (текст), `district`, `lat`, `lng`
- `area_total`, `area_living`, `area_kitchen`
- `rooms`, `floor`, `floors_total`
- `year_built`
- `building_type`: новостройка/вторичка/сталинка/хрущёвка/чешка/другое
- `condition`: дизайн-ремонт/евроремонт/жилое/под ремонт/без ремонта
- `description` (свободный текст)

**Опциональные (заполняет риелтор по желанию):**
- `heating_type` (отопление: централизованное/индивидуальное/газовое/электро)
- `balcony` (нет / балкон / лоджия / 2 и больше)
- `parking` (нет / во дворе / подземный / гараж)
- `furniture` (для аренды: пусто / частично / полностью)
- `appliances` (для аренды)
- `kids_allowed`, `pets_allowed` (для аренды)
- `deposit` (для аренды, число)
- `utilities_included` (для аренды, bool)
- `bathroom` (раздельный / совмещённый / 2 и более)
- `ceiling_height` (метры)
- `documents` (свидетельство / новостройка от застройщика / в собственности менее 3 лет)
- `complex_name` (название ЖК)

**Фото:** отдельная таблица `property_photos` (id, property_id, order, filename, is_cover).

### Риелтор (`agents`)
- `id`, `tg_id`, `name`, `position`, `experience_years`, `deals_count`
- `bio` (короткое описание)
- `phone`, `tg_username`, `languages` (массив)
- `photo`
- `is_active`

### Клиент (`clients`)
- `id`, `tg_id`, `first_name`, `last_name`, `phone`
- `created_at`, `last_active_at`

### Запись на просмотр (`viewings`)
- `id`, `property_id`, `client_id`, `agent_id`
- `scheduled_at`, `created_at`
- `client_name`, `client_phone` (снепшот, чтобы при удалении клиента осталось)
- `status`: `pending` | `cancelled_by_client` | `done`
- `reminder_day_sent`, `reminder_hour_sent` (bool)

### Избранное (`favorites`)
- `id`, `client_id`, `property_id`, `created_at`

### Курс (`exchange_rates`)
- `date`, `usd_uah`

### Аналитика просмотров (`property_views`)
- `id`, `property_id`, `client_id`, `viewed_at`

### Шеринг (`shares`)
- `id`, `property_id`, `client_id`, `created_at`

## 5. Логика записи на просмотр

1. Клиент жмёт «Записатися на перегляд» на детали объекта.
2. Форма: дата/время + имя + телефон (заполняется один раз в профиле, потом подставляется автоматически, редактируемо).
3. Правило: **минимум за 2 часа** до выбранного времени. Если меньше — кнопка disabled + текст «Оберіть час щонайменше за 2 години. Для термінового перегляду напишіть рієлтору напряму».
4. После записи: экран успеха «Дякуємо! Рієлтор зв'яжеться найближчим часом».
5. **Уведомление риелтору в бот:**
   ```
   🆕 Нова заявка на перегляд
   📍 {адреса}, {ціна}
   👤 {ім'я}, {телефон}
   📅 {дата/час}
   [Зв'язатися з клієнтом] [Скасувати]
   ```
6. **Напоминание клиенту в бот** (если у клиента есть взаимодействие с ботом):
   - За день до просмотра в 10:00
   - За час до просмотра
7. **Отмена клиентом**: кнопка «Скасувати перегляд» в Mini App → попап «Ви впевнені?» → отмена → уведомление риелтору:
   ```
   ❌ Клієнт скасував перегляд
   📍 {адреса}
   👤 {ім'я}, {телефон}
   📅 був на {дата/час}
   ```

|---------|-----------|

## 7. Связь клиент → риелтор

Кнопка **«Написати в Telegram»** на карточке объекта.
- Открывает `t.me/{agent_username}?text={шаблон}` через `Telegram.WebApp.openTelegramLink`
- Шаблон:
  > Доброго дня! Цікавить {тип об'єкту} за адресою {адреса}, ціна {ціна}. Чи можна дізнатися деталі?

Также видны:
- Имя + фото риелтора
- Должность, опыт, кол-во сделок
- Описание «о себе»
- Телефон (кликабельный → звонок)
- Языки общения

---

## 8. Шеринг объекта

Кнопка «Поділитися» в детали объекта.
- Использует `Telegram.WebApp.switchInlineQuery` или `openTelegramLink` с `https://t.me/share/url`
- Превью карточки (через настройку бота `inline_mode`): первое фото + цена + адрес
- Ссылка вида `https://t.me/{bot_username}/app?startapp=property_{id}` — открывает Mini App сразу на этом объекте

## 9. Аналитика (Q13 — кабинет + сводка в бот)

В Mini App «Кабінет ріелтора»:
- Активних об'єктів: N
- Зданих/проданих за місяць: N
- Нових заявок (неделя/месяц): N
- Конверсія: переглянули/в обраному/записались
- Найпопулярніший об'єкт тижня
- Усього клієнтів у застосунку: N

В бот раз в неделю — сводка тех же цифр.

## 10. Поиск (фронт)

- **Полнотекстовый**: по адресу + названию ЖК (без описаний)
- **Фильтры**: тип, сделка, цена от/до, комнаты, площадь, район, состояние, новостройка/вторичка
- **Карта**: те же фильтры применяются к точкам

---

## 11. Безопасность

- Валидация `initData` на каждом запросе к API (через HMAC от bot token)
- JWT с коротким TTL (1 час) + refresh
- Лимит на загрузку: 50 фото × 10 МБ = 500 МБ максимум на объект
- Rate limit на создание объектов и просмотров (защита от спама)
- Только риелторы (по TG ID из env) могут писать в `properties`, `agents`
